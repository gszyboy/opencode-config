请帮助用户将需求转化为结构化文档。

## 输入

用户的简单描述或一句话需求。

## 流程

### 第一步：理解需求

向用户提问澄清关键信息：
1. 核心用户是谁？
2. 核心功能有哪些？（按优先级 P0/P1/P2）
3. 需要哪些平台？（Web/小程序/H5）
4. 需要数据库吗？
5. 需要第三方集成吗？（微信登录、支付等）

### 第二步：生成需求文档

生成 `docs/requirements.md`，包含：

1. **模块划分**
   - 每个模块的名称、描述、优先级
   - 模块责任边界

2. **功能详细说明**
   - 用户故事
   - 操作流程
   - 界面元素
   - 业务规则
   - 验收标准

3. **技术规格**
   - 模块依赖关系
   - API 设计
   - 数据模型

4. **非功能需求**
   - 性能要求
   - 安全要求
   - 兼容性

5. **边界定义**
   - 包含什么
   - 不包含什么

### 第三步：生成开发计划

基于需求文档，调用 @plan-decomposer 生成 `docs/development-plan.md`

## 输出文件

- `docs/requirements.md` - 需求文档
- `docs/development-plan.md` - 开发计划

## 约束

- 保持简洁，不做过度设计
- MVP 只包含 P0 功能
- 每个模块独立可开发
