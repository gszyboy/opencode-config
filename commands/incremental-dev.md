## 任务：$ARGUMENTS

@~/.config/opencode/docs/existing-project-rules.md

请使用 50行平衡模式 执行任务：

## 改动约束
- 文件数：≤ 3
- 行数：≤ 50
- 不引入新依赖

## 改动前（必须执行）

1. **确认改动范围**
   ```
   本次任务：[用户描述]
   预计改动文件：
   - file1.py (约X行)
   预计不改动：[红线文件]
   ```

2. **检查红线**
   - 数据库结构/ORM模型 ❌
   - 认证鉴权逻辑 ❌
   - 支付财务代码 ❌
   - 核心业务算法 ❌

3. **自动备份**
   ```bash
   git tag "backup/before-$(date +%Y%m%d-%H%M%S)"
   ```

## 改动中
- 每次改动不超过20行
- 涉及2+文件时先说明改动关联
- 遇到红线立即停止，询问用户

## 改动后

1. **说明改动**
   ```
   已完成改动：
   - file1.py: [具体改动]
   
   验证方法：
   - [如何测试]
   ```

2. **验收清单**
   - [ ] 功能正常工作
   - [ ] 没有破坏现有功能
   - [ ] 改动在预算内

3. **更新变更记录**

   如果有变更需要记录，在 `docs/CHANGELOG.md` 中追加：

   ```markdown
   ### [日期] 变更 #N

   **发现**：[问题描述或调整说明]
   **调整**：[处理方式]
   **影响**：[影响范围]
   ```

   如果 CHANGELOG.md 不存在，创建它：

   ```markdown
   # 已有项目变更记录

   ## 🔧 变更记录

   ...
   ```

## 禁止事项
- ❌ 不重构无关文件
- ❌ 不"顺手优化"代码
- ❌ 不添加未要求的功能
- ❌ 不修改红线文件
