## 任务：查看开发进度

### 第一步：读取进度

读取 `docs/progress.json` 获取当前进度。

### 第二步：以友好格式展示

如果文件存在，以以下格式展示：

```
┌─────────────────────────────────────────────────────────────┐
│                    📊 项目开发进度                           │
├─────────────────────────────────────────────────────────────┤
│ 项目：{project}                                             │
│ 最后更新：{lastUpdated}                                      │
└─────────────────────────────────────────────────────────────┘

### 阶段 1：{phaseName}
进度：{completedTasks}/{totalTasks} ✅

[✅] 任务1：已完成
    完成时间：2026-03-22 14:30

[🔄] 任务2：进行中 ← 当前
    开始时间：2026-03-22 15:00
    备注：...

[⏳] 任务3：待开始
[⏳] 任务4：待开始
```

如果任务有 notes 字段，展示备注信息。

### 第三步：统计信息

展示：
- 总任务数
- 已完成数
- 进行中数
- 待开始数
- 完成百分比

### 第四步：建议

根据进度给出建议：
- 如果有进行中的任务：提示继续执行 `/dev`
- 如果所有任务都完成：提示进入下一阶段
- 如果有阻塞的任务：提示检查问题

### 如果 progress.json 不存在

提示用户：
```
进度文件不存在。
请先执行 /dev 开始开发任务，或检查 docs/development-plan.md 是否已创建。
```
