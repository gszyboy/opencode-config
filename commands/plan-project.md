请分析项目需求文档，生成详细的开发计划。

## 输入

首先查找项目中的需求文档：
- `docs/requirements.md`
- `docs/需求.md`
- `SPEC.md`
- `README.md`
- 或用户提供的内容

请指定需求文档位置，或直接提供需求内容。

---

## 工作流程

### 步骤 1：提取功能模块

从需求中提取核心功能模块：

```markdown
## 功能模块清单

### 模块 1：[模块名称]
- **优先级**：P0/P1/P2
- **功能描述**：
- **涉及页面/组件**：
- **API 需求**：
- **预计工时**：
- **前置依赖**：

### 模块 2：[模块名称]
...
```

### 步骤 2：分解为可执行任务

每个模块分解为具体任务：

```markdown
### 阶段 1：基础设施（1-2 天）
- [ ] 任务 1
- [ ] 任务 2

### 阶段 2：核心功能（3-5 天）
- [ ] 任务 3
- [ ] 任务 4

### 阶段 3：增强功能（2-3 天）
- [ ] 任务 5
```

### 步骤 3：优先级排序

| 优先级 | 说明 | 含义 |
|--------|------|------|
| P0 | 必须有 | 核心功能，无则项目不可用 |
| P1 | 应该有 | 重要功能，影响用户体验 |
| P2 | 可以有 | 锦上添花，可延后 |

## 约束

- MVP 只包含 P0 功能
- 每个任务 ≤ 4 小时
- 不超过 3 个开发阶段
- 中小型单体应用，不做微服务拆分
- 每个任务控制在 2-4 小时内

## 不要做的事

- ❌ 不要创建详细的数据库设计（初期只需要主要表）
- ❌ 不要设计完整的 API 文档（按需定义）
- ❌ 不要创建测试计划（后续单独做）
- ❌ 不要规划超过 3 个开发阶段

## 完成后

### 1. 保存开发计划

将开发计划保存到 `docs/development-plan.md`

### 2. 初始化 progress.json

根据开发计划生成 `docs/progress.json`，用于跟踪开发进度：

```json
{
  "project": "[项目名称]",
  "version": "1.0",
  "lastUpdated": "[当前时间 ISO 8601 格式]",
  "currentPhase": 1,
  "phases": {
    "1": {
      "name": "MVP",
      "status": "active",
      "tasks": [
        {
          "id": 1,
          "title": "任务1",
          "description": "任务描述",
          "status": "pending",
          "createdAt": "[当前时间]"
        }
      ]
    },
    "2": {
      "name": "完善功能",
      "status": "pending",
      "tasks": []
    },
    "3": {
      "name": "优化与增强",
      "status": "pending",
      "tasks": []
    }
  },
  "metadata": {
    "totalTasks": [总任务数],
    "completedTasks": 0,
    "inProgressTasks": 0,
    "pendingTasks": [总任务数]
  }
}
```
